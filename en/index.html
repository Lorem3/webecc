Secure Backup E2EE, Only You Can Access Your Data

How to Use

This page performs no network operations. All data is processed locally in your browser.

Encryption does not require a password. Only a passphrase is needed for decryption.

Backup

1. Enter passphrase

2. Enter backup email

3. Enter email subject

4. Passphrase prefix (can be an email or any text)

5. Generate bookmark link and save it

6. Open the page from the bookmark, enter content in the plaintext box, then click "Send encrypted result to email" below

For subsequent use, start from step 6 only. No need to re-enter the password.

Restore

1. Paste ciphertext in the input box below

2. Enter passphrase

3. Click "Generate from passphrase" to generate keys

4. Click Decrypt

Format specification

      
This page uses Curve25519 elliptic curve for encryption. All operations are performed locally.

Due to Curve25519 format requirements, see D. J. Bernstein's article for details. For example:
0DyqHFjCstGwvtppOuhTKDDQQdBOffta9X8Fu0TjEXQ=
1DyqHFjCstGwvtppOuhTKDDQQdBOffta9X8Fu0TjEXQ=
These actually correspond to the same key.

      
D. J. Bernstein

      …Computing secret keys. Inside your program, to generate a 32-byte Curve25519 secret key, start by generating 32 secret random bytes from a cryptographically safe source: mysecret[0], mysecret[1], ..., mysecret[31]. Then do
      
mysecret[0] &= 248;
mysecret[31] &= 127;
mysecret[31] |= 64;
