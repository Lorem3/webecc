CloudFlare D1 Backup Details

Store encrypted ciphertext in Cloudflare D1's free database, and restore it anytime via your bookmark URL.
All encryption is performed locally in the browser. CloudFlare only stores ciphertext and cannot access the plaintext.

Security

  • Content is encrypted with Curve25519 + AES-GCM-256 double-layer encryption before upload
  • The CloudFlare D1 owner, administrators, and any third party cannot decrypt or view the plaintext
  • Decryption requires both the private key / passphrase and SALT — either one alone is insufficient
  • CloudFlare only serves as a ciphertext storage medium and does not participate in encryption or decryption

Encryption Process

When saving to D1, plaintext undergoes two layers of encryption:

Layer 1: Curve25519 Encryption

The user's public key is used to Curve25519-encrypt the plaintext, producing ciphertext.
Decrypting this layer requires the corresponding private key (derived from the passphrase via PBKDF2).

Note: The private key and passphrase are not stored locally and are not included in the bookmark URL. Please keep them safe. If forgotten, stored data will be permanently lost and unrecoverable.

Layer 2: AES-GCM-256 Encryption

A content key derived from the public key and SALT is used to AES-GCM-256 encrypt the Layer 1 ciphertext.

Key Derivation

The content encryption key is derived through two-layer HMAC-SHA512:

PRK = HMAC-SHA512(key=pubkey, data=salt)
ContentKey = HMAC-SHA512(key=PRK, info="d1_content")[0..31]

The final ContentKey is truncated to the first 32 bytes, used as the AES-GCM-256 encryption key.

D1 Storage Key Derivation

The unique identifier key for D1 records is also derived via HMAC-SHA512:

PRK = HMAC-SHA512(key=pubkey, data=salt)
D1Key = HMAC-SHA512(key=PRK, info="cloudflare-d1-access")[0..32]

The final D1Key is truncated to the first 33 bytes, then Base64-encoded as the D1 record key.

AES-GCM Encryption Details

Parameter Value
Algorithm AES-GCM
Key length 256 bit (32 bytes)
IV length 12 bytes, randomly generated
Storage format IV (12 bytes) + ciphertext

Storage Format

Ciphertext stored in D1 is prefixed with N.. The overall structure is:

N. + Base64Encode(IV + AES-GCM-Ciphertext)

Where the AES-GCM-Ciphertext plaintext is the result of Curve25519 encryption.

Decryption Process

  1. Retrieve ciphertext from D1, identify the N. prefix
  2. Base64 decode, separate IV (first 12 bytes) and ciphertext
  3. Derive ContentKey using public key + SALT (same as encryption process)
  4. Use ContentKey + IV for AES-GCM-256 decryption, obtaining Curve25519 ciphertext
  5. Use user's private key for Curve25519 decryption, obtaining plaintext

Notes

  • Uses Cloudflare D1 free database, reliability is not guaranteed
  • Multiple records can be saved under the same public key + SALT, but subject to free D1 capacity limits. Some older data may be deleted later without prior notice
  • Please regularly export backups and do not rely solely on D1 storage
