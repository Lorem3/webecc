What is this for?

Periodically export encryption keys from your password manager as encrypted backups, and send them to your email.
All operations are performed locally in the browser. No server is involved.

I once lost the two-step verification for Bitwarden and had to delete my account.
Periodically export the manager's keys
Put the public key in the link for convenient backups each time, without re-entering the passphrase.

Format Specification

Structure

1 byte 1 byte 2 bytes 2 bytes 2 bytes IV MAC Random public key Encrypted data

All shorts below are little-endian

  • Byte 1: Format flag

    • Old format (Blake2b):
      • 4: gzip then encrypt (Blake2b)
      • 5: encrypt directly (Blake2b)
    • New format (SHA-512): Uses bit flags
      • bit3 (0x08): Always 1, indicates new format
      • bit2 (0x04): Always 1
      • bit1 (0x02): Reserved
      • bit0 (0x01): Compression flag, 0=gzip then encrypt, 1=encrypt directly
      • Example: 0x0C (12) = new format + compressed, 0x0D (13) = new format + uncompressed

  • Byte 2: Reserved (0)

  • Bytes 3-4: short value for IV length, 16
  • Bytes 5-6: short value for MAC hash length, 32
  • Bytes 7-8: short value for temporary public key length, 32
  • IV // Randomly generated
  • MAC // Computed and appended
  • tmpPubKey // Randomly generated public/private key pair, private key (tmpSecKey) is discarded after use
  • EncryptData

Process

Generate temporary key pair: tmpSecKey, tmpPubKey

sharedX (32 bytes) = publicKey * tmpSecKey

buffer[96];
copy sharedX => buffer[0..31]

/**
* Public keys are compared in little-endian order, high position first
*/
if tmpPubKey > publicKey
    copy publicKey => buffer[32...63]
    copy tmpPubKey => buffer[64...95]
else
    copy publicKey => buffer[32...63]
    copy tmpPubKey => buffer[64...95]
end

Select hash algorithm based on format:

【Old format (format=0)】
Use blake2b hash to generate 64-byte digest
digest = blake2b(buffer)

【New format (format=1)】
Use SHA-512 hash to generate 64-byte digest
digest = sha512(buffer)

digest[0..31] is used as AES-CBC key aesKey, along with IV to encrypt the key and produce EncryptData

digest[32,63] is used as MAC key HKEY, producing 32-byte output

【Old format MAC calculation】
Using blake2b, output 32 bytes, key = HKEY
mac = blake2b(IV + tmpPubKey + EncryptData, key=HKEY)

【New format MAC calculation】
Using HMAC-SHA-512, truncated to first 32 bytes
mac = hmac_sha512(HKEY, IV + tmpPubKey + EncryptData)[0..31]

Compatibility

  • Auto-detect format on decode: Determined by byte 1
    • byte0 = 4 or 5: Old format (Blake2b)
    • byte0 bit3 (0x08) = 1: New format (SHA-512)
    • Other values: Not supported, throws exception
  • Default to new format on encode: format parameter defaults to 1 (SHA-512)
  • Backward compatible: Can decrypt old format data, but new encryption defaults to SHA-512

Generating Keys from Passphrase

Steps

  1. Determine key based on bookmark ver and whether salt is included:
    • ver=2.0 (new format) and hash contains salt: key = passphrase (without prefix)
    • Old format (no ver, or no salt): key = prefix (if any) + passphrase
  2. PBKDF2(key) generates 32 bytes as private key, secKey = PBKDF2(key)
  3. Generate key pair from private key
